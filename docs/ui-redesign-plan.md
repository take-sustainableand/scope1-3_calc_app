# S&Carbon UI/UX リデザイン計画

- **作成日**: 2026-04-26
- **対象**: `index.html` / `styles.css` / `app.js` の3点で構成される SPA（Phase 1 完了時点）
- **ゴール**: 「グリーン × グレー」を主軸にしたスタイリッシュな配色とタイポグラフィに刷新し、サステナビリティ領域の個人ツールとしてのブランドを確立する
- **非ゴール**: 機能追加、データモデル変更、Phase 2 以降のロジック実装。今回はあくまで見た目と情報設計のブラッシュアップに限定する

---

## 1. デザイン方針

### 1.1 コンセプト

> **"Quiet Carbon Intelligence"** — 静かに、しかし確かに、炭素の輪郭を可視化する。

CO2 算定ツールは「数字をどれだけ信頼できるか」が体験の核。情報のヒエラルキーを過剰な彩度で取らず、**白に近いグレーの余白**と、**苔・モス・フォレストグリーンのアクセント**で語る方向に振り直す。

### 1.2 リブランディングの3原則

1. **Calm（鎮静）** — 飽和色・多色配色をやめ、グレースケール80%＋グリーン20% に統一する。Scope1/2/3 の色分けも「同じ緑系トーンの濃淡」で表現する。
2. **Precise（精密）** — 数値は等幅相当の数字フォント（Tabular Numerals）で揃え、桁の頭が縦に揃うようにする。
3. **Crafted（質感）** — 影・角丸・線幅を全画面で統一。`box-shadow` を弱め、`border` の階調で奥行きを出す方針に変える。

### 1.3 トーン＆マナー

| 項目 | Before | After |
| --- | --- | --- |
| 配色 | 青(primary) + 緑/紫/茶/赤 のマルチカラー | グレースケール + 単一グリーンの濃淡 |
| 角丸 | `--radius: 8px` 固定 | 4px / 8px / 16px の3段階で意味づけ |
| 影 | `0 16px 40px rgba(...)` の重い影 | `0 1px 0 rgba(0,0,0,.04)` + 1px 境界線で立体感 |
| 文字 | Inter 単独、太さ 600〜850 で過剰 | 見出しに Display 系、本文 Sans、数字 Tabular で役割分離 |
| 装飾 | 雲・ロゴのグラデーション多用 | フラット＋極小グラデのみ。アクセントは線で引く |

## 2. カラーシステム

### 2.1 新パレット（ライト）

ベースは「ウォームグレー」。冷たい青寄りグレーは使わず、ほんのり緑が透けるニュートラルにする。

| トークン | 値 | 用途 |
| --- | --- | --- |
| `--bg` | `#F4F5F2` | アプリ全体の背景（オフホワイト・ややグリーン寄り） |
| `--surface` | `#FFFFFF` | カード、モーダル |
| `--surface-soft` | `#F8F9F6` | テーブルヘッダ、入力背景、空状態 |
| `--surface-sunken` | `#EEF0EA` | セクション区切り、選択中行の背景 |
| `--line` | `#E2E5DE` | 細い境界線（カード枠、表の罫線） |
| `--line-strong` | `#C7CCC1` | 強い境界線（フォーカスリング、点線） |
| `--text` | `#1B1F1A` | 本文・見出し（純黒は使わない） |
| `--text-muted` | `#5A6157` | サブテキスト、ラベル |
| `--text-subtle` | `#8A9085` | キャプション、disabled |

### 2.2 グリーンスケール（Brand Greens）

単一系統の濃淡で Scope 別・KPI・状態色まで賄う。**多色を完全に排除する。**

| トークン | 値 | 用途 |

| `--moss-50` | `#EEF4EC` | バッジ背景、選択ハイライト |
| `--moss-100` | `#D7E4D2` | プログレス背景、極淡塗り |
| `--moss-300` | `#9DBA92` | 補助線、Scope 3 |
| `--moss-500` | `#5C8A4F` | プライマリ・Scope 2・主要グラフ |
| `--moss-700` | `#3D6634` | ホバー、Scope 1、見出しアクセント |
| `--moss-900` | `#1F3A1A` | ダーク文字色、ヒーロー背景 |

> **Scope 別色の置き換え方針**: 現在 `Scope1=青 / Scope2=緑 / Scope3=紫`。これを `Scope1=moss-700 / Scope2=moss-500 / Scope3=moss-300` の濃淡で表現する。識別はアイコン形状とラベルで補完する。

### 2.3 機能色（最小限）

| `--accent-info` | `#506B7A` | 情報（青をスレートに） |

### 2.4 ダークモード（Forest Night）

| --- | --- |

## 3. タイポグラフィ

### 3.1 フォント選定

GitHub Pages 配信＝外部CDN前提を避けたいので、**システムフォント優先＋限定的に Google Fonts** の二段構えを提案する。

| 役割 | 第一候補 | フォールバック | 採用理由 |
| --- | --- | --- | --- |
| Display（見出し） | `"Fraunces"` 540 wght, opsz 32 | `Georgia, serif` | 高めのコントラストとモダンセリフで「サステナビリティ系の上質感」を出す |
| UI Sans（本文・UI） | `"Inter"` 400/500/600 | `system-ui, -apple-system, "Hiragino Sans", "Yu Gothic UI", sans-serif` | 既存資産を活かしつつ字面の整い |
| Numeric（数値） | `"Inter"` の `font-variant-numeric: tabular-nums` | 同上 | 桁ぞろえ＋フォント追加コスト0 |
| 日本語 | `"Inter"` のあとに `"Hiragino Sans", "Yu Gothic UI", "Noto Sans JP"` | sans-serif | 日本語のウェイトを 400/500 のみに絞り、和欧混植のチラつきを抑える |

> **採用しないフォント**: Roboto / Helvetica。S&Carbon の語感に対してニュートラル過ぎる。

> **配信戦略**: Fraunces のみ Google Fonts から `display=swap` でロード。失敗時は Georgia にフォールバックさせ、UI が崩れない設計にする。

### 3.2 タイプスケール

8pt baseline grid の上に乗せる。`line-height` を本文 1.6、見出し 1.2 に分け、Display は文字サイズが大きいほどトラッキングを締める。

| トークン | size / line-height / weight / family | 用途 |
| --- | --- | --- |
| `--type-display-xl` | 40 / 44 / 540 / Fraunces, `letter-spacing: -0.02em` | オンボーディング・空状態のキャッチ |
| `--type-display-lg` | 32 / 38 / 540 / Fraunces, `-0.018em` | レポート扉、目標カードの大数字（Scopeトータル） |
| `--type-h1` | 24 / 30 / 600 / Inter, `-0.012em` | 画面タイトル（topbar h1） |
| `--type-h2` | 18 / 26 / 600 / Inter, `-0.008em` | カード見出し |
| `--type-h3` | 15 / 22 / 600 / Inter | サブ見出し、テーブルキャプション |
| `--type-body` | 14 / 22 / 400 / Inter | 本文 |
| `--type-body-strong` | 14 / 22 / 600 / Inter | 強調本文 |
| `--type-caption` | 12 / 18 / 500 / Inter, `0.02em` | ラベル、タグ |
| `--type-overline` | 11 / 16 / 600 / Inter, `0.12em`, uppercase | セクションラベル（"OVERVIEW" 等） |
| `--type-numeric-xl` | 44 / 48 / 540 / Fraunces, tabular-nums | KPIヒーロー |
| `--type-numeric-md` | 22 / 28 / 600 / Inter, tabular-nums | テーブル内の数値 |

### 3.3 タイポルール

- 数字は **必ず** `font-variant-numeric: tabular-nums` を当てる（KPI/テーブル/フォームのプレビュー）。
- `letter-spacing` は見出しほど締め、キャプションほど開く（光学的な整合）。
- 単位（kWh、t-CO2e）は本文ウェイトのまま、本数字より 60% のサイズ・`color: var(--text-muted)`。
- 日本語見出しに Fraunces は使わず Inter+和文ゴシックに統一。Display 系セリフは英数字キャッチコピーのみに限定する。
- 全体で **太さは 400 / 500 / 600 / 540(serif)** の4種に絞る。現在の 650/750/800/850 という細かい刻みは廃止する。

## 4. レイアウト・グリッド・スペーシング

### 4.1 8pt スペーシングシステム

現状の `padding: 18px` / `gap: 14px` などの不揃いな値を統一する。

| トークン | 値 | 主な用途 |
| --- | --- | --- |
| `--space-1` | 4px | 行内アイコン余白 |
| `--space-2` | 8px | チップ、バッジ内余白 |
| `--space-3` | 12px | フィールド間 |
| `--space-4` | 16px | カード間 gap |
| `--space-5` | 24px | カード内 padding（標準） |
| `--space-6` | 32px | セクション間 |
| `--space-7` | 48px | ページトップ余白 |
| `--space-8` | 80px | 空状態の上下余白 |

### 4.2 角丸スケール

| トークン | 値 | 用途 |
| --- | --- | --- |
| `--radius-sm` | 4px | バッジ、進捗バー |
| `--radius-md` | 10px | ボタン、入力、チップ |
| `--radius-lg` | 16px | カード、モーダル |
| `--radius-pill` | 999px | スイッチ、トグル |

> 既存の 6/8/9/22/26 など中間値は廃止。

### 4.3 影・境界線

- 影は **2段階のみ**：`--elev-1: 0 1px 0 rgba(27,31,26,0.04), 0 0 0 1px rgba(27,31,26,0.04)` / `--elev-2: 0 8px 24px -12px rgba(27,31,26,0.18)`
- カードはデフォルトで `border: 1px solid var(--line)` ＋ `--elev-1`。ホバーで `--elev-2`。
- ヒーローカード等の派手なグラデは廃止し、`--moss-900 → --moss-700` の極控えめな単色グラデのみ使う。

### 4.4 グリッド

- ダッシュボードの `kpi` グリッド `1.25fr repeat(3, 1fr)` は読みやすいが、ヒーローを **`span 6` の横長カード** に変更し、その下の Scope3つを 4-4-4 で並べる構造に再編する（数字主役の階層を強調）。
- `dashboard-mid` は現在 1.45 / 0.85 / 0.9 の3分割。これを **左 2/3：折れ線、右 1/3：縦積み（カテゴリ＋最近のアクティビティ）** に変えると視線移動が一方向で済む。
- 最大幅は `1280px` に詰める（現在 1440px）。広すぎる横幅で間延びしている画面が引き締まる。

## 5. コンポーネント刷新

### 5.1 ロゴ／ブランドマーク

- 現在の二重リーフ風グラデーションは「青⇄緑」が混ざって主張が強い。
- 提案: **モノクロのカーボン葉脈マーク** に置換。`--moss-700` 単色 + `--moss-300` のサブストロークで「葉脈」を描く SVG。アクセシビリティのためにテキストロゴ "S&Carbon" は Fraunces 540 + small caps 風の `font-variant-caps: all-small-caps` で組む。
- ファビコンも同 SVG から生成（背景 `--moss-900`、マーク `--moss-50`）。

### 5.2 ナビゲーション（左サイドバー）

- 現状: 半透明白背景＋青ホバー。
- 提案:
  - 背景を `--surface` 完全不透明、右に `1px solid var(--line)`。`backdrop-filter` は廃止（パフォーマンス/レンダリング揺れ防止）。
  - アクティブ項目は **左に 3px の `--moss-700` インジケータ** ＋ `background: var(--surface-sunken)`。文字色は `--text`、アイコンは `--moss-700`。
  - ホバーは `background: var(--surface-soft)` のみ。色は変えない。
  - `nav-button` の `min-height` を 40 → 44 に上げ、タップターゲットを確保。
  - 末尾の `side-note` は **「現在の期間 / 拠点」のサマリーピル** に変更し、フィルター状態を常時可視化する。

### 5.3 トップバー

- タイトル `h1` を `--type-h1`（24/30 Inter 600）に。サブタイトル（パンくず or 期間）を `--type-overline` で上に小さく置く。
- 右側のフィルター群は **入力ではなくチップ表現** に変える（クリックでメニューを開くタイプ）。`select` のネイティブUIはモバイルだけで使い、PC では同デザインのドロップダウンを提供する想定（実装はネイティブ select を CSS で偽装でも可）。
- アバターは廃止（個人利用ツールのため）。代わりに「同期状態 + 最終保存」を示す小チップを置く（例: `● ローカル保存 23:14`）。

### 5.4 ボタン

| 種別 | スタイル |
| --- | --- |
| Primary | 背景 `--moss-700`、テキスト `#fff`、ホバーで `--moss-900`、押下で 1px 内側シャドウ。グラデーション廃止。 |
| Secondary | 背景 `--surface`、`1px solid var(--line-strong)`、テキスト `--text`。ホバーで `border-color: --moss-700`。 |
| Ghost | 透明背景、テキスト `--moss-700`。ホバーで `background: --moss-50`。 |
| Destructive | テキスト `--accent-danger`、ホバーで `background: rgba(168,68,59,0.06)`。 |
| Icon-only | 36×36、`--radius-md`、ツールチップ必須。 |

- 全ボタン共通: `min-height: 40px`（モバイルは 44）、`font-weight: 600`、`letter-spacing: 0`、`gap: 8px`。
- フォーカスリングは `0 0 0 2px var(--surface), 0 0 0 4px var(--moss-500)` の二重輪で出す。

### 5.5 入力・フォーム

- ラベルは `--type-overline`（"AMOUNT" のような英字大文字）はやめ、**`--type-caption` の日本語ラベル** ＋ 必須マーク `*` を `--moss-700` で表示。
- 入力ボックスは `border-radius: --radius-md`、`min-height: 44px`、内側 padding `0 14px`。
- フォーカス時のみ `border-color: --moss-700` ＋ 2px の外側ハロー。
- エラーは赤枠＋下に 12px のヘルプテキスト（`--accent-danger`）。アイコンは ⚠ マークではなく「！」を細い丸で囲んだ独自の SVG。
- プレビュー（推定排出量）の数字は **Display 数字スタイル** で大きく表示し、入力の右側 or 下に固定パネル化する。

### 5.6 カード

- カードは `padding: 24px`（現在18）、見出しと本文の間に 16px の `gap`。
- カード見出し横の "詳細を見る" 系リンクは **アイコン付きの ghost ボタン** に統一し、矢印（→）アイコンを右に。
- `metric-card` のインナーは「ラベル → 数値 → デルタ → ミニ波形（任意）」の縦4段構成にし、Scope ごとに小さな波形（直近6ヶ月）を `--moss-300` で描く。

### 5.7 KPI / ヒーローカード

- 現在のヒーロー：青グラデーション＋雲のラジアルグラデは廃止。
- 提案：背景 `--moss-900`、テキスト `--moss-50`、右下にうっすら `--moss-700` の **大きなカーボン葉脈 SVG** を低彩度で配置。
- 大数字は Fraunces 540 / 56px / tabular-nums。単位は同じく `--moss-300` で 18px。
- 下段に「前月比デルタ」と「対目標進捗バー」をシンプルに。

### 5.8 グラフ

- 折れ線：色を `--moss-700` (Scope1) / `--moss-500` (Scope2) / `--moss-300` (Scope3) に変更。線幅は 2px、現在の 3px は太すぎ。
- グリッドラインは `--line`、ラベルは `--text-muted`。
- ドーナツ：5色マルチを廃し、**Scope1〜3 の3層**に絞る。残りは「未分類」として `--surface-sunken`。中央に総量＋単位を Display 表記。
- ツリーマップ：色をすべて `--moss` 系の濃淡で表現。テキスト色はマス色に応じて `--moss-50` / `--moss-900` を切り替える。
- 棒グラフ：`background: linear-gradient(90deg, var(--moss-700), var(--moss-500))` 一種類のみ。

### 5.9 バッジ・チップ

- 角丸 `--radius-pill`、`min-height: 22px`、`padding: 0 10px`、`--type-caption`。
- バリエーションを 3 種に絞る:
  - `badge--neutral`: `--surface-sunken` / `--text`
  - `badge--success`: `--moss-50` / `--moss-700`
  - `badge--warn`: `rgba(181,134,43,0.12)` / `--accent-warn`
- 既存の green/purple/amber/red の使い分けは廃止。Scope は **アイコン + 文字** で識別する。

### 5.10 テーブル

- `data-table` の `th` 背景を廃止し、**下に 1px の `--line-strong`** だけにする（よりエディトリアル）。
- 行間は 14px、罫線は `--line`、ホバー行のみ `--surface-soft`。
- 数値列は `text-align: right; font-variant-numeric: tabular-nums;` を当てる。
- ソート可能ヘッダにはアイコン（小さな三角）を `--moss-500` で。

### 5.11 トースト / モーダル / オーバーレイ

- トーストは右下→**左下**に変更（モバイルではタブバーと干渉しない上部固定）。
- 背景を `--surface` ＋ 左に 3px の `--moss-700` 帯。テキストは `--text`。
- モーダルの `box-shadow` は `--elev-2` のみ。半透明オーバーレイは `rgba(27,31,26,0.5)`。

### 5.12 モバイルタブバー

- 5タブ構成は維持。ただし `backdrop-filter` を廃止し `background: var(--surface)` 不透明にする（描画安定）。
- アクティブはアイコン色 `--moss-700` ＋ 上端 2px インジケータ。
- ラベルは `--type-caption`、ウェイト 500 に落とす。

## 6. 画面別ブラッシュアップ案

### 6.1 ダッシュボード（dashboard）

- ヒーロー：横長 1段、左に総排出量、中央に対目標プログレス、右に「今月の入力件数」。
- 下段：左 2/3 に折れ線（Scope 別 3線）、右 1/3 に縦積みで「カテゴリ構成（ドーナツ）」「最近のアクティビティ」。
- "クイックアクション" は3つ → 4つへ拡張可（データ入力 / 原単位 / レポート / 目標）し、アイコン主体のタイル型に。
- オンボーディング画面は Display 見出しで "Track your carbon, quietly." のような英字キャッチを置き、3ステップカードを**横並び 3列**に。

### 6.2 データ入力（data-input）

- 左：入力フォーム、右：**スティッキープレビューパネル**（既存）を維持しつつ、プレビューを「数式表現」で見せる:

  ```
  電力（日本） 0.000434 t-CO2e/kWh
  × 7,039,474 kWh
  ─────────────
  3,055.13 t-CO2e
  ```

  数字を Fraunces 大型でレイアウトすると、計算過程の納得感が一気に上がる。
- 既存の `input-table` は **片側1列**にレイアウト変更し、各行の右端に削除アイコン。
- 入力エラー時は行ごと `--surface-sunken` で囲って強調。

### 6.3 原単位管理（factors）

- リストは現在の3列 (`48px / 1fr / auto`) を維持しつつ、左 48px をアイコンタイル化（`--moss-50` 背景＋`--moss-700` アイコン）。
- 係数の数字を Fraunces 大型に。単位 `t-CO2e/kWh` は `--text-muted` 12px。
- 検索バーは上部固定。フィルターチップ（Scope1/2/3、カテゴリ、ステータス）を横スクロール可能なバーに。

### 6.4 排出量分析（analytics）

- 現在「ダーク前提」の画面だが、**ライト時もダーク時も統一テーマ**で組む（多モード分岐の負債を減らす）。
- 12カラムグリッドを活かし、上段：折れ線（span 8）+ ドーナツ（span 4）、中段：ツリーマップ（span 7）+ 拠点別バー（span 5）、下段：表（span 12）。
- カラーリングはすべて moss スケール。凡例はテキスト＋ドット、ドット直径 8px。

### 6.5 レポート（reports）

- "扉" を Display スタイルで作る：年度・期間・拠点を Fraunces 32px で大書し、下に CSV/JSON 出力ボタン。
- 月次サマリー、Scope 別明細、原単位マスタを **3 つのカードに分割**し、各カードの右上に「コピー」「ダウンロード」アイコン。

### 6.6 目標・進捗（goals）

- カード内の数値ペアを `現状 → 目標` のフレーズで Display フォント表示（例: `1,250 → 875 t-CO2e`、間に細い矢印）。
- プログレスバーは 8px → 6px に細くし、`--moss-700` のフィル＋背景 `--moss-100`。閾値超過時のみ `--accent-danger`。

### 6.7 削減施策（actions）

- 3列カンバンを維持。各カラムヘッダに件数バッジ。
- カードは `padding: 14px`、上に Scope バッジ、下に担当・期限。**ドラッグ操作は将来課題**として記載（現状はクリックでステータス変更）。

### 6.8 アラート（alerts）

- 重要度別の色遊びを廃止し、左に 3px の縦バーで重要度を示す（`--moss-700` / `--accent-warn` / `--accent-danger`）。
- 各アラートに「対応する」「無視する」のセカンダリボタンを 1 行に。

### 6.9 設定（settings）

- 3列のセッティンググリッドは維持。各カードに `--type-overline` でセクション名。
- GitHub トークン欄は **専用カード**として分離し、上部に「ローカル保持・送信されません」の注意書きを `--moss-50` のインフォボックスで表示。
- テーマ切替は3択（Light / Dark / System）。プレビュー画像も moss トーンに更新する。

### 6.10 データ一覧（data-list）

- 現在のテーブル＋検索 UI を維持しつつ、**選択行のサマリーフッター**を画面下部に出す（合計排出量、件数）。
- 行クリックでサイドモーダル（右からスライドイン）を開き、編集導線を統一。

## 7. アイコノグラフィ

- **線幅 1.5px の単一スタイル**に統一。現在 `stroke-width: 2` で厚ぼったいので 1.5 に。
- 角は `stroke-linecap: round; stroke-linejoin: round;` を維持。
- セット候補：Lucide / Phosphor (regular)。ライセンス的に再配布も容易。プロジェクトには SVG をインライン埋め込みで取り込む（外部依存ゼロを維持）。
- Scope アイコンは独自に作成：
  - Scope 1: 工場と火（直接燃焼）
  - Scope 2: 稲妻と矢印（電力購入）
  - Scope 3: 三角形が連なる流れ（バリューチェーン）
- ファビコン・OG画像も moss スケールで再制作する。

## 8. モーション・マイクロインタラクション

ベース時間: `--motion-fast: 120ms`、`--motion-base: 200ms`、`--motion-slow: 320ms`。
イージング: `--ease-standard: cubic-bezier(0.2, 0, 0, 1)`、`--ease-emphasized: cubic-bezier(0.3, 0, 0, 1)`。

| 要素 | モーション |
| --- | --- |
| 画面遷移 | `#app` 内をクロスフェード 200ms。`prefers-reduced-motion` 時は無効。 |
| カードホバー | `transform: translateY(-1px)` + 影遷移 `--motion-fast`。 |
| ボタン押下 | `transform: scale(0.98)` 80ms。 |
| プログレスバー | 値変更を 320ms の `--ease-emphasized` で補間。 |
| トースト | 下から 8px スライドイン＋フェード 200ms、4 秒後にフェードアウト。 |
| 数値カウントアップ | KPI ヒーローのみ、ロード時に 600ms で 0 → 値へ補間（`requestAnimationFrame` で実装）。 |

`prefers-reduced-motion: reduce` を尊重し、上記すべてを 1ms に短縮するクラスを root に当てる。

## 9. アクセシビリティ

- すべてのテキスト/背景の組み合わせで WCAG AA（本文 4.5:1、Display 3:1）を満たす。`--moss-500` on `--surface` は 4.6:1 を確認済み（プレビュー値）。
- フォーカスリングは前述の二重輪。`:focus-visible` のみで描画（マウスクリックでは出さない）。
- アイコンのみのボタンは必ず `aria-label`。
- `data-table` には `caption` を付与し、ソートボタンは `aria-sort` を更新。
- カラーのみで意味を伝えない（Scope の色は同系統濃淡なので、必ずアイコン＋テキスト併用）。
- フォーム必須項目に `aria-required` と `aria-describedby` を設定。
- ダーク/ライトの自動切替を尊重する `prefers-color-scheme` を初期値に採用しつつ、設定での上書きを保持。

## 10. パフォーマンス・実装上の留意点

- フォントは `Fraunces` のみ Google Fonts、`text=Sans=swap`。CSS 内に `@font-face` を直書きし、ファイルサイズを抑えるサブセット（英数字・記号のみ）を選択。
- `backdrop-filter` 廃止でモバイル GPU 負荷が下がる。
- `innerHTML` 全置換による再描画コストは現状維持。ただし**カード単位で文字列を分割し、配列 join に統一**しておくと、後で部分更新へ移行しやすい。
- アイコン SVG は `defs` で一度だけ定義し、`<use>` で参照することでバンドルサイズを削減。
- CSS は `:root` のトークン定義を上に集約し、その下を「リセット → レイアウト → コンポーネント → ユーティリティ → メディアクエリ」の順で再構成する。

## 11. 段階的リファクタリング計画

設計をいきなり全面差し替えするとリグレッションのリスクが高いので、**5 フェーズ**に分割する。各フェーズは 0.5〜1 日想定。

### Phase A: トークン基盤の差し替え（破壊的変更なし）

1. `:root` の CSS 変数を新パレット・新スペーシング・新角丸トークンに差し替え。
2. 既存クラス名は維持。色トークン名（`--primary`, `--green` など）は **エイリアス** として `--moss-500` 等を指す形で残し、後方互換を保つ。
3. ダークモードの `--bg / --surface / --text` を Forest Night に更新。
4. 動作確認: `node --check app.js`、`xmllint --html --noout index.html`、ローカルサーバで全画面巡回。

**成功定義**: 既存 UI が崩れず、全体トーンがグリーン×グレーに切り替わる。

### Phase B: タイポグラフィ刷新

1. `index.html` に Fraunces の `<link>` を追加（`preconnect` 含む）。
2. `:root` に `--type-*` トークンを追加。
3. `h1/h2/h3/.metric-value/.calc-number` を Display スタイルに変更。
4. 数字に `font-variant-numeric: tabular-nums` を一括適用（`.metric-value`, `.calc-number`, `.factor-value`, `.data-table td.is-numeric`）。
5. ボタン・ラベルのウェイトを 600 統一に整理。

**成功定義**: 数字の桁が縦揃い、見出しと本文の階層が視覚的に明確化する。

### Phase C: コンポーネントスタイル刷新

1. ボタン4種、カード、入力、バッジ、テーブル、トースト、モーダルを順に更新。
2. ヒーローカード・ドーナツ・折れ線の色を moss 系に置換。
3. 影をすべて `--elev-1 / --elev-2` の二段に統一。
4. ロゴマークを SVG 差し替え。

**成功定義**: 全画面が同一のコンポーネント語彙で構成され、配色のばらつきがなくなる。

### Phase D: 情報設計（画面別レイアウト）刷新

1. ダッシュボードのヒーローを横長 1段に再構築。
2. 入力プレビューを「数式表現」に。
3. 分析画面のテーマをライト/ダーク統一に。
4. レポート画面の扉ページを実装。

**成功定義**: ダッシュボード・入力・分析のメインフローで「数字を読みやすく、操作が一直線」な体験になる。

### Phase E: モーション・アクセシビリティ仕上げ

1. `--motion-*` トークン追加 + 各遷移にトランジションを当てる。
2. `prefers-reduced-motion` 対応。
3. `:focus-visible` リング、`aria-*` 属性の付与。
4. WCAG AA コントラストの最終チェック（DevTools の対比チェッカ or `axe`）。

**成功定義**: キーボード・スクリーンリーダ・モーション低減ユーザーで主要操作が完了できる。
